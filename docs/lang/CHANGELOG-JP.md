## [1.1.0] – 2025-10-18

### ✨追加しました

- **スマートな段落グループ化エンジン**
  元の Markdown レイアウトを維持しながら、複数行の段落をまとまったブロックとして変換するためのインテリジェントな段落グループ化が追加されました。
  → 行揃えを変更せずに、より正確でコンテキストを認識した翻訳を生成します。

- **差分プレビュー モード (実験的)**
  ユーザーは、出力を保存する前に翻訳の差分をプレビューできるようになりました。
  VS コード出力パネルの `auto-translate-readmes.previewDiff` コマンド。

- **README 文字セットの自動検出**
  クロスプラットフォームの一貫性を保つために、ソース ファイルのエンコーディング (`UTF-8`、`UTF-16`、または `UTF-8-BOM`) を自動的に検出して保存します。

---

### 🔧 変更されました

- **インライン マークダウン トークナイザー v3**
  複雑なマークダウンの組み合わせを保護するためにトークナイザーを更新しました。

- `**bold + _italic_ mix**`
- `[Click _here_](url)`
- `<span>` や `<img>` などのインライン HTML 要素

- **コード ブロックの一貫性**
  コード ブロックは、元のインデントとスペースで完全に保持されるようになりました (トリミングや余分な改行はありません)。

- **言語スイッチャーの書式設定の改善**
  多言語スイッチャーは、ターゲット言語ごとに句読点とスペースのスタイル (日本語の全角コロン、フランス語のコロンの前のスペースなど) を自動的に調整するようになりました。

- **より安定した Google API 統合**
  `translate.googleapis.com` がレート制限またはネットワーク エラーに達した場合のセカンダリ エンドポイントへの自動フォールバックが含まれるようになりました。

---

### 🐞修正済み

- **大胆な取り扱い (エッジケース)**
  `**Before:**` または `**After (Absolute):**` が `* *Before:**` に分割されるケースを修正しました。

- **リスト内の非改行スペース**
  リスト項目内のハイフン `-` が一部の言語 (フランス語やロシア語など) のテキストと直接結合される問題を防止しました。

- **末尾の改行を保持**
  翻訳出力では、ソース ファイルと同じ数の空白行が保持されるようになり、何も削除されません。

- **絵文字の安定性**
  翻訳中に絵文字や特殊文字がドロップされるエンコードの問題を修正しました。

- **クロスプラットフォームの EOL の一貫性**
  ソース ファイルに従って一貫した行末文字 (Windows の場合は `\r\n`、Unix の場合は `\n`) を保証します。

---

### 🚀 開発者向けノート

- 変換ロジックは `translateBlock()` でモジュール化され、非同期バッチ処理により最大 30% 高速な変換が可能になりました。
- 生成された各 README にはメタデータ コメントが含まれています。
  `<!-- Auto-Translated v1.1.0 -->`
  ビルド バージョンを自動的に追跡します。
- **VS Code Marketplace v1.1 安定版リリース**の準備ができています。

## [1.0.9] - 2025-10-17

### 変更されました

- **洗練された翻訳エンジン (最終改訂版)**
  元の `README.md` の正確な構造、間隔、改行数を保持するように翻訳プロセスを完成させました。
- 改行保持ロジック (`split(/(\n)/)`) を導入し、翻訳されたすべてのファイルで同一のレイアウトを保証します。
- リスト、表、コード ブロック、見出し、リンク、画像の翻訳を安全にスキップできるよう Markdown 認識が改善されました。
- ソース ファイルとの 1:1 行マッピングを維持するために、書式設定の正規化 (改行のトリミング、正規表現の間隔など) を削除しました。
- 混合コンテンツ行 (インライン コード、絵文字、またはリスト マーカーを含むテキスト) の処理が強化されました。
- 一貫した UTF-8 エンコーディングと保持された EOL 形式 (Windows `\r\n` または Unix `\n`) を使用して出力が書き込まれることを保証します。

＃＃＃ 修理済み

- **正確なマークダウン忠実度** – 翻訳されたファイルは、ソース `README.md` と視覚的および構造的に正確に一致するようになりました。
- **インデントの復元** – サブリストまたはネストされたリスト項目が翻訳後にインデントを失う問題を修正しました。
- **空行の保存** – 段落とリストの間で消える空行を修正しました。
- **太字およびインライン コードの処理** – 行頭の `**bold**` および `inline code` の形式が正しくないケースを修正しました。
- **テーブル行の整合性** – 非ラテン言語 (中国語、韓国語など) にわたる複雑な Markdown テーブルの破損した区切り線を修正しました。
