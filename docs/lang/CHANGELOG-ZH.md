## [1.1.0] – 2025-10-18

### ✨ 已添加

- **智能段落分组引擎**
添加了智能段落分组，可将多行段落转换为内聚块，同时保留原始 Markdown 布局。
→ 在不改变行对齐的情况下生成更准确、上下文感知的翻译。

- **差异预览模式（实验性）**
用户现在可以在保存输出之前预览翻译差异
VS Code 输出面板中的 `auto-translate-readmes.previewDiff` 命令。

- **自动检测自述文件字符集**
自动检测并保留源文件编码（`UTF-8`、`UTF-16` 或 `UTF-8-BOM`）以实现跨平台一致性。

---

### 🔧 已更改

- **内联 Markdown Tokenizer v3**
更新了分词器以保护复杂的 Markdown 组合：

- `**bold + _italic_ mix**`
- `[Click _here_](url)`
- 内联 HTML 元素，如 `<span>` 或 `<img>`

- **代码块一致性**
代码块现在完美保留了原始缩进和间距（没有修剪或额外的换行符）。

- **改进的语言切换器格式**
多语言切换器现在会自动调整每种目标语言的标点符号和间距样式（例如日语全角冒号、法语冒号前的空格）。

- **更稳定的 Google API 集成**
现在包括当 `translate.googleapis.com` 达到速率限制或网络错误时自动回退到辅助端点。

---

### 🐞 已修复

- **大胆处理（边缘情况）**
修复了 `**Before:**` 或 `**After (Absolute):**` 被拆分为 `* *Before:**` 的情况。

- **列表中的不间断空格**
防止了列表项中的连字符 `-` 直接与某些语言（例如法语或俄语）的文本合并的问题。

- **保留尾随换行符**
翻译输出现在保留与源文件相同数量的空行 - 不会修剪任何空行。

- **表情符号稳定性**
修复了翻译过程中表情符号或特殊字符丢失的编码问题。

- **跨平台 EOL 一致性**
根据源文件确保行尾字符一致（Windows 为 `\r\n`，Unix 为 `\n`）。

---

### 🚀 开发者笔记

- 翻译逻辑现在在 `translateBlock()` 下模块化，具有异步批处理，​​可将翻译速度提高约 30%。
- 每个生成的自述文件都包含元数据注释：
`<!-- Auto-Translated v1.1.0 -->`
自动跟踪构建版本。
- 准备好 **VS Code Marketplace v1.1 稳定版**。

## [1.0.9] - 2025-10-17

### 已更改

- **精炼翻译引擎（最终修订版）**
完善了翻译过程，以保留原始 `README.md` 的_精确_结构、间距和换行数。
- 引入了换行符保留逻辑 (`split(/(\n)/)`)，确保所有已翻译文件的布局相同。
- 改进了 Markdown 识别，可以安全地跳过列表、表格、代码块、标题、链接和图像的翻译。
- 删除了格式规范化（例如，换行符修剪、间距正则表达式）以保持与源文件的 1:1 行映射。
- 增强了对混合内容行（带有内联代码、表情符号或列表标记的文本）的处理。
- 确保输出使用一致的 UTF-8 编码和保留的 EOL 格式（Windows `\r\n` 或 Unix `\n`）编写。

＃＃＃ 固定的

- **精确的 Markdown 保真度** – 翻译后的文件现在在视觉上和结构上与源 `README.md` 完全匹配。
- **缩进恢复** – 修复了子列表或嵌套列表项在翻译后丢失缩进的问题。
- **空行保留** – 修复了段落和列表之间消失的空行。
- **粗体和内联代码处理** – 修复了行开头的 `**bold**` 和 `inline code` 格式错误的情况。
- **表格行完整性** – 修复了非拉丁语言（例如中文、韩语）的复杂 Markdown 表格中损坏的分隔线。
